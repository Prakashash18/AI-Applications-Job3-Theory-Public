18CNN recap—not a reteach

Same CNN engine.
New data strategy.

Rescale→Conv + Pool→Conv + Pool→Conv + Pool→Dense vote

19Read the evidence

Two datasets. Two signals.

Training accuracy / loss

How well the model fits the examples used to update it.

Validation accuracy / loss

How well the model performs on held-out examples.

1Direction

2Gap

Are training and validation far apart?

3Stability

Does validation improve or become worse?

Do not judge from training accuracy alone. The validation line tells us whether learning transfers to new data.

21Part 2 · Apply augmentation

Build a small pipeline of random transformations.

The transformations are grouped in a Sequential block so each training image can pass through them in order.

1RandomFlipmirror horizontally
2RandomRotationtilt within a range
3RandomZoomchange framing
data_augmentation.py
data_augmentation = keras.Sequential([
    layers.RandomFlip(
        "horizontal",
        input_shape=(image_height, image_width, 3)
    ),
    layers.RandomRotation(0.1),
    layers.RandomZoom(0.1),
])
22Code → plain meaning

CodePlain-language jobWhat students should notice

24Build the improved model

Two changes fight overfitting.

↻

Data augmentation

Makes the input practice more varied.

data_augmentation
◌

Dropout

Prevents the network from relying too heavily on particular activations.

layers.Dropout(0.2)
Order: augment → rescale → CNN feature layers → dropout → dense output.
data_augmentation.py
model = Sequential([
    data_augmentation,
    layers.Rescaling(1./255),
    layers.Conv2D(16, 3, padding='same', activation='relu'),
    layers.MaxPooling2D(),
    layers.Conv2D(32, 3, padding='same', activation='relu'),
    layers.MaxPooling2D(),
    layers.Conv2D(64, 3, padding='same', activation='relu'),
    layers.MaxPooling2D(),
    layers.Dropout(0.2),
    layers.Flatten(),
    layers.Dense(128, activation='relu'),
    layers.Dense(number_of_classes, name='outputs')
])

img = tensorflow.keras.utils.load_img(
    test_path, target_size=(image_height, image_width)

29Interpret the output

Confidence is a score—not proof.

The code applies softmax to convert output scores into values that add up to 100%, then reports the largest one.

sunflowers78%
dandelion12%
tulips6%
roses3%
daisy1%
78%

The model strongly prefers “sunflowers”.

It can still be wrong if the image is unusual, blurry, outside the known classes or unlike the training data.

Good practice: inspect the image and prediction together—especially when the confidence is low or surprising.
Teacher promptAsk: “Can a model be confidently wrong?” Expected answer: yes. Confidence describes the model’s score, not guaranteed truth.
